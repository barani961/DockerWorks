Spring Boot Application Docker Setup

This project demonstrates how to containerize and run a Spring Boot application using Docker.

Docker allows the application to run in an isolated environment with all required dependencies.

1. Build the Docker Image

Build the Docker image from the Dockerfile in the current directory.

docker build -t my-spring-app .

Explanation

docker build → builds an image from the Dockerfile
-t my-spring-app → assigns a name (tag) to the image
. → uses the current directory as the build context

2. Run the Docker Container

Run a container using the created image.

docker run -p 8080:8080 my-spring-app

Explanation

docker run → creates and starts a container
-p 8080:8080 → maps host port 8080 to container port 8080
my-spring-app → the image name

After running the container, the application will be available at:

http://localhost:8080

3. Dockerfile Used in This Project

FROM eclipse-temurin:21-jdk-alpine

ARG JAR_FILE=target/*.jar

COPY ${JAR_FILE} app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "/app.jar"]

Explanation

FROM → base image containing Java runtime
ARG → defines the location of the compiled JAR file
COPY → copies the JAR into the container
EXPOSE → exposes port 8080
ENTRYPOINT → command used to start the application

4. Docker Installation Check

Check if Docker is installed.

docker --version

5. Docker Image Commands

5.1 Build an image from a Dockerfile

docker build -t image_name .

Example

docker build -t todo-app .

This builds a Docker image using the Dockerfile in the current directory.

5.2 Pull an image from Docker Hub

docker pull <image_name>

Example

docker pull nginx

This downloads an image from Docker Hub.

5.3 List all locally stored images

docker images

or

docker image ls

Displays all images stored locally.

5.4 Push an image to Docker Hub

docker push <username>/<image_name>

Uploads an image to Docker Hub.

5.5 Remove a local image

docker rmi <image_name>

Deletes an image from the local system.

5.6 Search for images on Docker Hub

docker search <image_name>

Searches for images available in Docker Hub.

6. Docker Container Commands

6.1 List running containers

docker ps

Shows only running containers.

6.2 List all containers

docker ps -a

Shows running and stopped containers.

6.3 Run a container

docker run image_name

Creates and runs a container from an image.

6.4 Run container in interactive mode

docker run -it image_name

Starts a container with an interactive terminal.

6.5 Run container in detached mode (background)

docker run -d image_name

Runs the container in the background.

6.6 Run container with a custom name

docker run -d --name container_name image_name

Creates and runs a container with a specific name.

6.7 Map host port to container port

docker run -p host_port:container_port image_name

Example

docker run -p 8080:80 nginx

This maps port 8080 on the host to port 80 inside the container.

6.8 Stop a running container

docker stop container_name

Stops a running container.

6.9 Start a stopped container

docker start container_name

Starts an existing container.

6.10 Restart a container

docker restart container_name

Restarts a container.

6.11 View container logs

docker logs container_name

Displays logs generated by the container.

6.12 Execute a command inside a running container

docker exec -it container_name bash

Runs a command inside a running container.

7. Docker Volume Commands

7.1 List volumes

docker volume ls

Lists all Docker volumes.

7.2 Create a volume

docker volume create volume_name

Creates a new Docker volume.

7.3 Remove a volume

docker volume rm volume_name

Deletes a Docker volume.

8. Docker Cleanup Commands

8.1 Remove unused images

docker image prune

Removes dangling images.

8.2 Remove unused containers, networks, images, and cache

docker system prune

Cleans up unused Docker resources.

9. Copy JAR File Into a Running Container

docker cp target/NAME_OF_JAR.jar CONTAINER_NAME:/tmp

Explanation

docker cp → copies files between host and container
target/NAME_OF_JAR.jar → compiled JAR file location
CONTAINER_NAME:/tmp → destination inside the container

10. View Files Inside Container

Example command

docker exec -it boring_chandrasekhar ls /tmp

Explanation

docker exec → runs a command inside a running container
-it → interactive terminal
boring_chandrasekhar → container name
ls /tmp → lists files in the /tmp directory inside the container
